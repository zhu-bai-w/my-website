# 正文标题

游戏系统拆解的重点不是把所有规则抄下来，而是理解这些规则如何共同塑造玩家体验。

## 先看玩家目标

分析时我会先判断玩家在当前阶段想完成什么，以及系统如何让这个目标变得明确。

## 再看行动与反馈

一个机制是否清楚，往往取决于玩家能不能快速理解行动结果。

### 三个观察点

- 行动是否有明确入口。
- 反馈是否及时可见。
- 失败是否能带来学习。

## 观察循环

把行动、资源、风险、奖励和成长连起来，就能看到游戏的核心循环。

## 回到体验

拆解最终要回到玩家感受：紧张、掌控、好奇、成就或叙事代入。

## 方法沉淀

每次拆解结束后，我会把问题框架、优秀做法和可迁移启发整理成笔记。
